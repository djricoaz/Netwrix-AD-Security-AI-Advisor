# Netwrix-AD-Security-AI-Advisor
Netwrix AD Security AI Advisor is a local Windows portal that transforms PingCastle Active Directory assessments into AI-enriched, easy-to-understand, actionable reports mapped to relevant Netwrix solutions for technical and non-technical stakeholders alike.

This project integrates with PingCastle but is not a redistribution of PingCastle free. PingCastle is subject to its own license terms.” That aligns with the PingCastle repo/license language and commercial-use guidance.  ￼
## Purpose

**Netwrix AD Security AI Advisor** is a local Windows portal designed to:

* run **PingCastle** against an Active Directory domain
* import and analyze **XML / HTML** reports generated by PingCastle
* enrich the assessment with **local AI**
* translate results for non-technical audiences
* correlate findings with the **most relevant Netwrix products**
* generate readable reports for security teams, management, and internal stakeholders

The goal is to provide a **clear, actionable, and visual view** of an Active Directory assessment without requiring users to read raw PingCastle files.


---

## What the tool does

The tool provides the following capabilities.

### Execution and import

* run PingCastle directly from the portal
* import existing PingCastle reports
* support for **XML** and **HTML** report files

### Analysis

* detailed parsing of PingCastle findings
* analysis by infrastructure area
* local AI interpretation
* translation of the content into the selected language
* exposure of evidence and coverage without opening raw files

### Presentation

* executive dashboard
* detailed findings view
* technical view
* MITRE ATT&CK mapping
* Netwrix product recommendations
* finding-to-product correlation graph

### Product correlation

The tool correlates findings with multiple Netwrix solutions and returns:

* **Primary product**
* **Secondary product**
* **Supporting products**

The tool does not automatically force **Netwrix Auditor** to appear first. It is intended to propose the **best fit** based on the nature of the findings.

# Troubleshooting

### The portal is not reachable

Check:

* that the **Netwrix AD Security AI Advisor** service is running
* that port `8510` is listening
* that the browser is opening `http://localhost:8510`

Useful command:

netstat -ano | findstr 8510

### The portal service stops immediately

Typical causes:

* incorrect EXE packaging
* wrong entry point
* missing dependencies or paths

Actions:

* review the service logs
* test the EXE manually from the installed folder

### The Ollama service does not start

Typical causes:

* port `11434` already in use
* another Ollama instance is already running

Useful command:

netstat -ano | findstr 11434

### The analysis returns no results

Check:

* that PingCastle actually generated the output files
* that XML/HTML parsing is available
* that an AI model is selected
* that the services are running correctly

### PDF/Word reports are unreadable

Recent builds are expected to improve:

* column widths
* text wrapping
* MITRE tables
* validation checks
* landscape layout where needed
